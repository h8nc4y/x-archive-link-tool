作成: 2026-07-03 JST（Claude Code Fable 5、codex-deep GPT-5.5 相談を反映）
位置づけ: `docs/post-release-operations-decision-packet.md`（正本）の8項目を、**公開範囲の3モード**別に縮退させた決定支援資料。オーナーが「モードを1つ選ぶ＋決定票に記入する」だけで Issue #42 を決着できる形を目指す。
本資料は判断材料であり決定ではない。最終決定はオーナーが行い、結果は `docs/post-release-human-verification-template.md` の形式で記録する。

## 使い方

1. §1 でモードを1つ選ぶ（迷ったら「自分専用」から開始し、後で昇格してよい）。
2. §2 の該当列を既定値として、変えたい項目だけ上書きする。
3. §3 の決定票に記入し、Issue #42 へ貼る（または本ファイルに追記して commit する）。

## 1. モード定義

| モード | 想定 | Issue #42 の重さ |
| --- | --- | --- |
| **M1: 自分専用** | URLは実質個人用。他者への案内なし | 最小（多くを「N/A: 個人運用」で決着） |
| **M2: 限定共有** | 知人・関係者に URL を教える | 中（privacy/support/429の最低限だけ決める） |
| **M3: 一般公開** | 不特定多数への公開・告知 | 全項目の正式決定が必要（現行 decision packet 通り） |

前提（全モード共通・変更しない）:
- X API は任意BYOT、既定は oEmbed-first（2026-02 X API有料化後の現実解）
- 本文・実URL・postId をログに残さない現行の安全境界は維持
- 魚拓はサーバー取得せず外部リンク提示のみ

## 2. 8項目 × 3モードの既定値案

### OPS-PRIVACY（privacy/legal 文言）
- **M1**: 現行 `privacy.html` のまま。法務レビューなし。「個人運用のため法務最終版としない」と記録して決着。
- **M2**: 現行文言に「個人運営・ベストエフォート」の明記を追加。法務レビューなし。
- **M3**: 法務観点レビュー（人間/ChatGPT）を実施し最終版化。

### OPS-SUPPORT（支援窓口）
- **M1**: 窓口なし（N/A）。
- **M2**: 連絡先は共有相手に直接伝達（サイト非掲載）。対応はベストエフォート。
- **M3**: 公開窓口（GitHub Issues または専用メール）を決定・掲載。エスカレーション経路を定義。

### OPS-BILLING（X APIクレジット監視）
- **M1/M2**: BYOTを設定しない限り N/A。設定する場合のみ「月上限額」と「確認頻度（月1回）」をオーナーが数値で決める。
- **M3**: 担当者・確認頻度・停止条件（上限額到達で `X_BEARER_TOKEN` を外し oEmbed-only へ縮退）を正式決定。

### OPS-LOGS（ログ保持）
- **M1**: Cloudflare 既定のまま。追加の保持・レビュー運用なし。
- **M2**: 同上＋障害時のみ確認する方針を記録。
- **M3**: 保持期間・削除方針・レビュー担当を正式決定し `privacy.html` に反映。

### OPS-429（rate limit 方針）
- **M1**: 本番429確認はしない。ローカルテスト（実装済み）で足りるものとして決着。
- **M2**: 同上。共有相手に「連続利用で待たされることがある」旨を口頭/メモで案内。
- **M3**: 本番429の実挙動確認の要否を判断（実施なら別承認）。ユーザー向け文言は実装済みのものを正とする。

### OPS-SMOKE（本番smoke）
- **M1**: 実施しない（自分の通常利用が事実上のsmoke）。
- **M2**: 実施しない、または最大1回（`docs/production-smoke-runbook.md` の承認文言・記録制限に従う）。
- **M3**: runbook 通りの承認・回数上限・記録で実施。

### OPS-KV（インシデント/キャッシュ障害）
- **M1/M2**: インシデントオーナー=本人。KV障害時は「キャッシュなしで継続、直らなければ利用を止める」で決着（`docs/incident-and-kv-failure-runbook.md` を簡易参照）。
- **M3**: オーナー・継続基準・ロールバック承認経路を正式決定。

### OPS-DATA（運用記録の境界）
- **M1**: 追加運用記録を作らないため N/A。
- **M2/M3**: 運用記録を作る場合のみ、記録者=オーナー、既存 redaction ルール（抽象値のみ）を適用。
