# Bounded Implementation

- 対象ファイルを絞り、指定外のファイルを不用意に変更しない。
- 仕様外改善、広範なリファクタ、不要な依存追加、フォーマットのみの変更を避ける。
- 既存挙動、公開API、ドキュメント上の約束を壊さない。
- 不明な仕様は補完せず `未確認` と書く。
- checkpoint は作業整理単位であり、完了だけを停止理由にしない。
- ローカルで安全に継続できる改善が残る場合は、次の小さな差分へ進む。
